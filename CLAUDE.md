## Architecture

Static showcase website for Betonea, a Serbian decorative concrete manufacturer. Next.js 16 App Router with SSG ‚Äî no backend, no database. All content lives in TypeScript data files.

**Key constraint:** This is Next.js 16.2.2. `params` in page components and `generateMetadata` is a **Promise** ‚Äî must `await params` before use. Read `node_modules/next/dist/docs/` for breaking changes.

### Data Flow

All content is static TypeScript in `src/data/`:
- `site.ts` ‚Äî contact info, phones, WhatsApp, social links, stats (imported everywhere)
- `categories.ts` ‚Äî 14 product categories with slugs matching route params
- `products.ts` ‚Äî ~177 products with real ≈Ýifra codes, EUR prices, dimensions. Helper functions: `getProductsByCategory()`, `getProduct()`, `getFeaturedProducts()`
- `gallery.ts` ‚Äî gallery images organized by category slug
- `testimonials.ts` ‚Äî 3 client testimonials

Types defined in `src/types/index.ts`: `Category`, `Product`, `Testimonial`, `GalleryImage`.

### Routes

Serbian-language URLs. All statically generated via `generateStaticParams`:
- `/` ‚Äî homepage (6 sections)
- `/proizvodi` ‚Äî product catalog (14 categories)
- `/proizvodi/[slug]` ‚Äî category page (slug = category.slug from categories.ts)
- `/proizvodi/[slug]/[productSlug]` ‚Äî product detail (productSlug = product.slug from products.ts)
- `/galerija` ‚Äî gallery with category filter
- `/o-nama` ‚Äî about page
- `/kontakt` ‚Äî contact with EmailJS form + Google Maps
- Custom 404 via `not-found.tsx`

### Images

Product and gallery images load from **external** `betonea.com/wp-content/uploads/` via `next.config.ts` remotePatterns. Category cover images and hero are local in `public/images/`.

**Cached image fix:** All `<img>` tags use a `ref` callback checking `img.complete` on mount because `onLoad` doesn't fire for browser-cached images. This pattern appears in ProductCard, CategoryCard, ProductGallery, GalleryGrid.

### Styling

**Tailwind CSS v4** ‚Äî configured via CSS `@theme inline` in `globals.css`, NOT a `tailwind.config.ts` file. Custom tokens:
- Colors: `stone-50..900`, `gold-50..600`, `cream`, `charcoal`
- Fonts: `font-heading` (Playfair Display), `font-body` (Inter)

### Layout

Header is `fixed` with hero-aware logic: detects `section.-mt-16` elements on mount to determine if page has a hero. Pages WITH hero get transparent header with white text; pages WITHOUT hero get solid white header with dark text. Hero components use `-mt-16 lg:-mt-20` to extend behind the fixed header; `main` has `pt-16 lg:pt-20` to compensate.

### Component Organization

- `ui/` ‚Äî reusable primitives (Button, SectionHeading, PageHero, Breadcrumb, ScrollReveal)
- `layout/` ‚Äî Header, Footer, MobileMenu, WhatsAppButton
- `home/` ‚Äî homepage sections (Hero, Services, FeaturedCategories, Testimonials, etc.)
- `products/` ‚Äî ProductCard, CategoryCard, ProductGallery, InquiryButton
- `gallery/` ‚Äî GalleryGrid, GalleryFilter, Lightbox
- `contact/` ‚Äî ContactForm (EmailJS), Map
- `about/` ‚Äî Stats (animated counters), ClientLogos

### SEO

- `generateMetadata` on every page with OpenGraph tags
- JSON-LD: LocalBusiness on homepage, Product schema on product detail pages
- Sitemap generated by `next-sitemap` on postbuild
- `robots.txt` in public/

### Contact Mechanisms

Three channels on product pages via InquiryButton: mailto (pre-filled), WhatsApp (wa.me), tel: link. Utils in `src/lib/utils.ts`: `createMailtoUrl()`, `createWhatsAppUrl()`, `formatPhoneForTel()`.

EmailJS form on `/kontakt` ‚Äî service/template IDs in `ContactForm.tsx` need real values before production use.

## Language

All user-visible text is in Serbian (Latin script) with proper diacritics: ƒá, ƒç, ≈°, ≈æ, ƒë. Never use ASCII equivalents.
